## What is Schorle?

**Schorle** is a hybrid Python/TypeScript framework that bridges the gap between Python backend development and modern frontend user interfaces. Build full-stack applications using Python for business logic while leveraging React/TypeScript for UI components, all within a unified development experience.

```python
from fastapi import FastAPI
from myapp.ui import Index

app = FastAPI()

@app.get("/")
def home():
    return Index(
        title="Welcome to Schorle",
        user_count=1234
    )
```

```jsx
import { useParams } from "schorle";

export default function Index() {
  const { title, userCount } = useParams();

  return (
    <div>
      <h1>{title}</h1>
      <p>Join {userCount} developers</p>
    </div>
  );
}
```

## Key Features

- **Zero Frontend Setup** - No webpack, no build scripts, no configuration hell
- **Type-Safe Integration** - Automatic TypeScript bindings from Python
- **Server-Side Parameters** - Pass data from Python to React components
- **Auto-Generated APIs** - Client-side hooks generated from FastAPI endpoints
- **Modern Stack** - React, Tailwind CSS, and Shadcn UI included
- **Single Process** - Embedded JavaScript runtime, no separate servers

## The Problem Schorle Solves

Python developers face suboptimal choices for modern web UIs:

- **Server-side rendering** (FastAPI + Jinja2) - Limited interactivity
- **WebSocket frameworks** (Streamlit/Dash) - Poor scaling, limited flexibility
- **Separate codebases** (FastAPI + React) - Complexity and duplication
- **Pure JavaScript** (Next.js) - Loss of Python ecosystem

**Schorle's approach:** Keep Python for what it's best at (backend logic, data, ML) while leveraging React for modern UIs, connected through sophisticated type-safe IPC.
